Case · Cross-device UX

FainTV

跨 TV · Mobile · Web 的訂閱體驗設計

台灣 OTT 電視盒 + Mobile / Tablet / Web

1M+ 用戶｜家庭與年長觀眾｜訂閱制 + 商城

My Role — UI/UX Designer

2 Designers · 1 PM · 3 Engineers

FainTV 是台灣影音串流，客廳電視盒為主，也有手機與網頁。訂閱加商城是營收核心。我負責跨三端 UX：同一套會員與訂閱體驗，在不同裝置上一致且各自好用。

Symptoms

當時看到的問題

訂閱流程流失嚴重
各裝置體驗不一致
TV 端任務完成率僅 30%

脈絡：TV 主力用戶約 60 歲

重設計前：訂閱走不完、各端體驗不一致、TV 完成率約三成。不一致不只是「長得不一樣」，而是任務與標準沒被統一。我們知道有問題，但還不能直接下解法。

Design question 如何用「一套跨裝置設計」，讓觀看與訂閱任務在對的人、對的裝置上完成？
先問卷再訪談。洞察不是單修 TV 表單，而是角色與任務跨裝置錯位。需要統一的任務模型與帳號關係，不是三端各自為政。

System

同一產品 · 三端設計

同一套帳號與任務模型，在 TV、Mobile、Web 用各自正確的方式表達

TV

= 看／驗證

焦點 · 電話驗證｜操作者：觀看者

Mobile

掃卡訂閱 · 完成後電視同步｜操作者：常為子女

Web

= 管

消費紀錄 · 權益｜操作者：本人或家人

統一

帳號關係 · 任務模型 · 視覺語言

分化

輸入方式 · 資訊密度 · 操作者

不是三套產品，是同一套體驗的三種表達

核心是跨裝置統一設計。三端同一會員與訂閱關係。統一：帳號、任務切分、產品語言。分化：輸入、密度、操作者。Mobile 完成後電視同步＝跨裝置狀態一致。不是 RWD，是任務模型 + 每端正確表達。

TV · Deep dive

TV — 同一帳號在 10-foot 的表達

Focus單一焦點系統
Identity認證／登入（同一身分體系）
States錯誤與狀態設計
A11yWCAG 可感知／可操作
TV 不扛完整結帳，但登入驗證必須在同一帳號體系。焦點與可讀性是 TV 端標準；與 Mobile／Web 共享規則、入口不同。

Mobile · Deep dive

Mobile — 訂閱主場 · 跨裝置閉環

掃卡 · 原生鍵盤 完成頁：電視已同步啟用
Mobile 承接高輸入交易；完成回寫 TV，三端同一訂閱結果。這是統一設計閉環。

方案後用易用性測試驗證跨裝置路徑，再迭代。完成率與時間來自測試；營收是業務結果。被問歸因時承認還有其他因素，我負責體驗與完成路徑。

跨裝置一致的，是心智模型與設計標準，不是介面。

TV 看／驗證 Mobile 訂 Web 管 操作者甚至不是同一個人
一致性定在任務、角色與標準，不是像素。看多裝置、多角色產品（含官網 B2C／B2B）也是：同一套設計邏輯的正確表達，不是同一個版面複製。
