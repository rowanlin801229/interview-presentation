Case · Cross-device UX

FainTV

跨 TV · Mobile · Web 的訂閱體驗設計

台灣 OTT 電視盒 + Mobile / Tablet / Web

1M+ 用戶 · 家庭與年長觀眾 · 訂閱制 + 商城

My Role — UI/UX Designer

2 Designers · 1 PM · 3 Engineers

FainTV 是台灣影音串流，客廳電視盒為主，也有手機與網頁。訂閱加商城是營收核心。我負責跨三端 UX：同一套會員與訂閱體驗，在不同裝置上一致且各自好用。

Main Issues

TV 端任務完成率僅 30%

Key context TV 主力用戶約 60 歲

重設計前：訂閱走不完、四種裝置無法流暢體驗、TV 完成率約三成。問題不只是畫面不一樣，而是跨裝置任務與標準沒被統一。我們知道有問題，但還不能直接下解法。主力用戶是六十歲左右的年長觀眾。

Insights

「訂閱的人，往往不是看電視的人」

1

遙控器輸入卡號與個人資料根本不可能。一個繁體中文字要填好送出至少需按十次以上，把自己名字輸入完就已經累了。

2

登入流程的 email 根本行不通——年長使用者根本沒有使用 email 的習慣。→ 改用電話號碼登入

3

訂閱常由子女在手機／電腦代完成

Design question 如何用「一套跨裝置設計」，讓觀看與訂閱任務在對的人、對的裝置上完成？
洞察來自訪談：遙控器輸入卡號與個人資料幾乎不可能，繁中一個字就要按十次以上，光打名字就累了；登入用 email 也行不通，年長用戶沒有 email 習慣，因此改電話登入；訂閱常由子女代完成。需要跨裝置任務模型，不是單修 TV 表單。

System · 01 / TV

同一產品 · 三端設計

TV = 看／驗證

10-foot 環境：電話註冊、大字級、遙控器可完成的最少步驟

會員基本資料 · 焦點框 · 遙控器操作｜操作者：觀看者

TV 端承接「看」與身分驗證，不硬塞完整結帳。電話帳號與全裝置同一套會員體系；錯誤與焦點狀態為 10-foot 設計。

System · 02 / Mobile

Mobile 訂閱流程

Mobile = 訂 · 子女代訂主場 · 七步完成

高輸入任務放在觸控裝置；完成後電視同步啟用

掃卡 · 原生鍵盤 完成頁：家中電視已同步啟用
Mobile 七步：資料、資訊、住址、付款、掃卡、確認、完成。操作者常為子女；完成頁明確告知長輩無需再操作。

System · 03 / Web

Web = 用戶複雜資訊管理

可收視／消費紀錄 · 兩端比較

Web 端提供詳細的用戶資訊

TV · 消費紀錄

遙控器 · 單列焦點

Web · 消費紀錄

滑鼠 · 完整表格
Web 端提供詳細的用戶資訊與完整帳務管理；TV 則是精簡查閱。同一任務、不同表達。
